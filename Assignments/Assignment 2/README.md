# Finance Lab Case Study

## Overview:
This assignment focuses on analyzing financial data using advanced SQL concepts such as joins, subqueries, and CTEs.
The goal is to identify unusual account activity and analyze customer behavior patterns.

## Database Used:
Financelab

## Tables:
* Customers
* Accounts
* Transactions

## Case Studies:

### Case Study 1: Silent High-Value Accounts:
Identify accounts with high balance but low or irregular transaction activity.
A calculated field **Account_flag** is used to describe abnormalities.

### Case Study 2: Hidden Customer Behavior Network:
Analyze customer activity patterns across different cities.
A calculated field **Customer_behavior** is used to classify customers.

## Concepts Used:
* INNER JOIN
* LEFT JOIN
* RIGHT JOIN
* SELF JOIN
* Subqueries
* Common Table Expressions (CTE)
* Aggregate Functions (SUM, COUNT, AVG)

## Language:
SQL
